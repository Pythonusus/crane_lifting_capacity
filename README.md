<a name = "about"></a>
## 📋 Описание

**Калькулятор грузоподъемности строительных кранов** - это веб-приложение для расчета грузоподъемности и коэффициента запаса строительных кранов. Приложение предоставляет удобный интерфейс для инженеров, проектировщиков и специалистов в области строительной техники.

### 🚀 Основные возможности

- **Двухрежимный расчет грузоподъемности**:
  - Расчет максимального веса груза (payload) при заданном коэффициенте запаса
  - Расчет коэффициента запаса при заданном весе груза
- **Выполнение сравнительного анализа кранов по техническим и экономическим показателям**
- **История расчетов** - автоматическое сохранение всех расчетов в локальное хранилище (до 50 записей) с возможностью просмотра, экспорта и удаления
- **Глубокая фильтрация кранов** - интеллектуальный поиск самого дешевого крана, способного поднять заданный груз на указанном вылете стрелы
- **База данных кранов** - обширная база данных строительных кранов различных производителей с полными техническими характеристиками
- **Продвинутая фильтрация и поиск**:
  - Фильтрация по модели, производителю, типу шасси, стране
  - Фильтрация по диапазону максимальной грузоподъемности
  - Подбор кранов по вылету и весу груза
  - Сортировка по различным параметрам
- **Пагинация с подгрузкой** - эффективная загрузка данных с использованием паттерна "load-more"
- **Экспорт и копирование результатов** - выгрузка отчетов по расчетам в различных форматах и копирование результатов в буфер обмена
- **API интерфейс** - RESTful API для интеграции с другими системами
- **Проверка лицензий** - интеграция с системой лицензирования Midas для контроля доступа
- **Современный UI** - адаптивный веб-интерфейс с интуитивным дизайном на базе Semantic UI React

<a name = "quick-start-в-docker-compose"></a>
## 🐳 Quick Start в Docker Compose

### Требования

- Docker
- Docker Compose
- Make (опционально, для удобства)

### Запуск приложения

1. **Клонируйте репозиторий:**
```bash
git clone https://github.com/Pythonusus/crane_lifting_capacity.git
cd crane_lifting_capacity
```

2. **Выберите необходимую версию приложения**
- Переключитесь на ветку develop для запуска последней dev версии
```bash
git checkout develop
```
- Оставайтесь в ветке main для запуска последней стабильной версии

3. **Создайте файл .env (опционально):**
```bash
cp .env.example .env
# Данный пункт можно пропустить
# Все переменные окружения имеют дефолтные значения для оптимальной работы приложения
# Подробнее про настройку окружения смотри в разделе "Переменные окружения" ниже
```

4. **Подгрузите предзаполненную базу данных в ```crane_lifting_capacity/data/app.db.sqlite3```

5. **Запустите приложение:**
```bash
# =====================
# С использованием Make
# =====================
make docker-start

# ======================
# Без использования Make
# ======================

# Создать директорию для логов с необходимыми разрешениями
# для использования как bind mount для логов контейнера
mkdir -p ./app_logs && chmod 777 ./app_logs

# Запустить приложение
docker compose up
```

6. **Откройте браузер:**
```
http://localhost:8000
```

<a name = "переменные-окружения"></a>
## 🔧 Переменные окружения
Все переменные имеют оптимальные дефолтные значения,
для работы приложения дополнительная конфигурация не требуется.

Подробнее смотри в .env.example в корне проекта

| Переменная | Описание | Значение по умолчанию |
|------------|----------|----------------------|
| `PORT` | Порт для запуска приложения | `8000` |
| `TZ` | Часовой пояс | `Europe/Moscow` |
| `DEVELOPMENT` | Режим разрабтчика | `false` |
| `LICENSE_CHECK` | Проверять ли лицензию пользователя | `true` |
| `DATA_DIR` | Путь к директории с базой данных крана | `./data` |
| `LOGS_DIR` | Путь к директории логов | `./app_logs` |
| `CORS_ORIGINS` | Разрешенные CORS источники | `*` |
| `CORS_CREDENTIALS` | Разрешить CORS credentials | `true` |
| `CORS_METHODS` | Разрешенные HTTP методы | `*` |
| `CORS_HEADERS` | Разрешенные HTTP заголовки | `*` |
| `PAGINATION_SIZE` | Размер страницы для пагинации | `30` |
| `DATABASE_URL` | URL для подключения к базе данных | `sqlite:///crane_lifting_capacity/data/app.db.sqlite3` |

<a name = "api-endpoints"></a>
## 🌐 API Endpoints

### Основные endpoints

| Метод | Endpoint | Описание |
|-------|----------|----------|
| `GET` | `/` | Главная страница приложения |
| `GET` | `/health` | Проверка состояния приложения |
| `GET` | `/docs` | Swagger UI документация |
| `GET` | `/redoc` | ReDoc документация |
| `GET` | `/metrics` | Prometheus метрики |

### Расчеты

| Метод | Endpoint | Описание |
|-------|----------|----------|
| `POST` | `/process` | Расчет грузоподъемности крана. Поддерживает два типа расчетов:<br/>- Расчет максимального веса груза (payload) при заданном коэффициенте запаса<br/>- Расчет коэффициента запаса при заданном весе груза |


### Управление кранами

| Метод | Endpoint | Описание |
|-------|----------|----------|
| `GET` | `/api/cranes/{crane_name}` | Получить кран по названию |
| `POST` | `/api/cranes` | Получить отфильтрованный список кранов с пагинацией (load-more). Поддерживает фильтрацию по модели, производителю, типу шасси, стране, диапазону грузоподъемности, а также глубокую фильтрацию по радиусу и весу груза |
| `POST` | `/api/cranes/count` | Получить количество кранов в БД по фильтрам |
| `GET` | `/api/chassis-types` | Получить доступные типы шасси |
| `GET` | `/api/manufacturers` | Получить список доступных производителей |
| `GET` | `/api/countries` | Получить список доступных стран производителей |
| `GET` | `/api/sort-options` | Получить доступные опции сортировки |
| `GET` | `/api/attachments/{attachment_id}` | Получить вложение крана (технические документы, изображения) |


<a name = "архитектура"></a>
## 🏗️ Архитектура

Приложение построено на следующем стеке технологий:

- **Backend**: FastAPI (Python 3.13) с SQLAlchemy ORM + Alembic
- **Frontend**: React, Semantic UI React, Vite
- **База данных**: SQLite с предзаполненными данными
- **Контейнеризация**: Docker и Docker Compose
- **Мониторинг**: Prometheus метрики и health checks
- **Лицензирование**: Интеграция с Midas license checker для контроля доступа
- **Хранилище**: LocalStorage для сохранения истории расчетов на клиенте
